Assert

Compares a boilersuit expression against a value and takes a specific action if the assertion is not true.

Any syntax or fatal error occurring during assertion will be handled as a fatal error and the script will be stopped. Assertion will be automatically false.

Syntax

ASSERT {bs-expression} = {value} ELSE {action}

{action} may be either STOP or WARN

Special features

  • If {bs-expression}'s result is a table or view, the first column of the first row will be compared to the expected value

Examples

      -- stop script if mytable has more or less than 3 rows
      ASSERT #mytable = 3 ELSE STOP;
  
      -- the previous one is equivalent to...
      ASSERT SELECT COUNT(*) FROM mytable = 3 ELSE STOP
  
      -- ...but it could be anything (here we would check first col of first row):
      ASSERT mytable = 3 ELSE STOP

NativeSQL

BoilerSuit understands a subset of the native SQL as long as it is understood by the underlying database system. Currently, the following commands are supported:
  • SELECT
  • CREATE TABLE [AS]
  • DELETE
  • INSERT
  • UPDATE

Syntax

Samples

myvariable := SELECT FROM mytable WHERE x = 2;

ChangeDirectory

Change or prints out current working directory

Syntax

CD {path}; -- to change directory CD; -- to print directory

Special features

Depending on operating system, syntax of path may vary. On Windows, backslash has been seen working; on linux slash.

After (successful) change of working directory, Boilersuit will automatically open the data.db file as current database.

Export

Exporting supported files.

Syntax

variablename =: [filename [delim(delimcharacter)][quote(quotecharacter)][replacedelim(replacestring)][replacequote(replacestring)]]
Extensions will be interpreted as follows:
  • csv - exports as csv with default options
  • xls - exports as Microsoft (R) Excel 1997-2003
  • xlsx - exports as Microsoft (R) Excel

Samples

-- as simply as it can get - exports ALL variables
=:

existing_table_name =: file_name.xml;
existing_table_name =: file_name.xls;
existing_table_name =: file_name.xlsx;
existing_table_name =: file_name.csv;
existing_table_name =: file_name.csv with delim(";"), quote("\'"), replacedelim(","), replacequote("_");

On the use of delim, quote, etc. (Simple)

To export using ';' as a delimitator (instead of the default ','), you would use

existing_table_name =: file_name.csv with delim(";")

You could also use tabs, like so:

existing_table_name =: file_name.csv with delim("\t")

If you are in conflict with values using the desired symbols, use

-- replace e.g. O'Brien by O\'Brien not to come in conflict with quotes '
existing_table_name =: file_name.csv with delim(";"), quote("'"), replacequote("\\'")

On the use of delim, quote, etc. (Advanced)

Argument of function delim() will be used as a delimitator to separate the strings in csv file. Argument of function quote() will be used as a quote to wrap the strings in csv file. Argument of function replacedelim() will be used for replace all of occurrences of delimitator in exported strings. Argument of function replacequote() will be used for replace all of occurrences of quote in exported strings.

Match

Takes care of the match commands.

Essentially, you have two data sets with entries that you know should more or less match, but not all of the entries do so exactly (respectively, it is hard or even impossible to find a good key for the exact matching of records). BoilerSuit helps you to make a fuzzy matching based on text similarity measures.

Usage: MATCH table_x(attribute1) ON table_y(attribute2) MATCH table_x(attribute1) ON table_y(attribute2) WITH threshold(0.9) MATCH table_x(attribute1, attribute2) ON table_y(attribute3, attribute4) ______________________________________________________________________________ THRESHOLD is used for different fuzzy match strategies, such as Jaro, Dice etc.

FrequencyDifference

Compares the frequency distribution of attributes. More information is above the {@link #process() method}

Usages: Only Attribute:

official.claim_id ./. previous.claim_id;

Whole Table:

official(claim_id) ./. previous(claim_id);

In the example above, there is a table (or view) official, containing dc_claim_id plus other attributes. There is also a table (or view) previous, containing claim_id plus other attributes.

Whole Table behave in the same way as the "only attribute" case, but in all lines, in addition to the attributes dc_claim_id, claim_id and the counts, we will add ALL attributes of the original tables, i.e. all official attributes + all previous attributes.

Map

To apply mappings to an attribute of a table, you should manually create a mapping table. containing all the unique values of the one attribute.

Usage: map table_name(attribute1) use mapping_table_name

table_name will be saved into {@link #originalTable} attribute1 will be saved into {@link #originalTableAttribute} mapping_table_name will be saved into {@link #mappingTable}

Mapping table should look like: "source", "target" "mercedes", "Boring_German_Car" "lada", "Racing_Machine"

Column 'source' contains values from original table. Column 'target' contains values used for replace the values from original table, so in above example all occurrences of value 'mercedes' will be replaced with 'Boring_German_Car' and all occurrences of value 'lada' will be replaced by 'Racing_Machine'.
